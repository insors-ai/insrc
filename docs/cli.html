insrc CLI Reference

Local-first hybrid coding agent — command reference

Quick Start

insrc                          # Start interactive REPL
insrc "explain the auth flow"  # One-shot question
insrc ask "refactor utils.ts"  # Classify + execute + exit
insrc plan "add caching layer" # Generate implementation plan
insrc daemon start             # Start background indexer
insrc repo add .               # Register current repo

Default Command — REPL

insrc [message] [--cwd <path>]

Start the interactive REPL agent. If message is provided, processes it as a single turn and exits.

ArgumentRequiredDescription
messageoptionalOne-shot question or task
insrc                                    # Interactive REPL
insrc "how does the indexer work?"       # One-shot, then exit
insrc --cwd /path/to/repo               # REPL in specific repo

insrc ask

insrc ask <message> [--intent <name>] [--claude] [--json] [--cwd <path>]

Classify intent, execute one turn, print result, and exit.

Daemon Commands

insrc daemon start

insrc daemon start

Start the background indexer/search daemon. The daemon:

  • Parses and indexes repos via tree-sitter (TypeScript, Python, Go)
  • Stores entities in Kuzu (graph) and LanceDB (vector)
  • Watches for file changes, re-indexes incrementally
  • Delta-indexes on startup (mtime-based) for files changed while stopped
  • Listens on ~/.insrc/daemon.sock for JSON-RPC

insrc daemon stop

insrc daemon stop

Gracefully stop the running daemon.

insrc daemon status

insrc daemon status

Display daemon health: uptime, queue depth, model readiness, and registered repos.

Repo Commands

insrc repo add <path>

insrc repo add <path>

Register a repo for indexing. Triggers a full index on first registration.

insrc repo remove <path>

insrc repo remove <path>

Unregister a repo and remove all its graph/vector data.

insrc repo list

insrc repo list

List registered repos with status and last indexed time.

Statuses: pending, indexing, ready, error

Agent Commands

insrc agent list

insrc agent list   (alias: insrc agent ls)

List all agent runs. Shows run ID, agent type, status, last update, and repo. Auto-detects crashed runs.

insrc agent resume <runId>

insrc agent resume <runId>

Resume a paused or crashed run from its last checkpoint.

insrc agent discard <runId>

insrc agent discard <runId>

Delete a run and all its artifacts.

insrc agent prune

insrc agent prune [--days <n>]

Remove completed runs older than n days (default: 7).

Conversation Commands

insrc conversation compact

insrc conversation compact [--repo <path>] [--hot-days <n>] [--warm-days <n>] [--cold-days <n>] [--dry-run]

Run tiered compression on stored conversation turns.

OptionDescriptionDefault
--repo <path>Scope to one repoall repos
--hot-days <n>Days to keep verbatim7
--warm-days <n>Days before cold compression30
--cold-days <n>Days before archive90
--dry-runPreview without applyingfalse
Compaction tiers: Hot (<7d, verbatim) → Warm (7-30d, truncated) → Cold (30-90d, merged) → Archive (>90d, summary). User directives ("never use opus") are preserved across all tiers.

insrc conversation stats

insrc conversation stats [--repo <path>]

Display conversation storage statistics: total turns, sessions, breakdown by type/tier/repo.

Test Commands

insrc test run <files...>

insrc test run <files...> [--review] [--framework <name>] [--kind <type>] [--timeout <ms>] [--cwd <path>]

Run the full tester pipeline: analyze → plan → write → execute → fix.

OptionDescriptionDefault
--reviewEnable code review gates per test filefalse
--frameworkOverride: vitest, jest, pytest, goauto-detect
--kindunit or liveunit
--timeoutPer-test timeout (ms)60000

Non-interactive — auto-approves all gates. Exits 0 if all pass, 1 otherwise.

insrc test run src/agent/planner/steps.ts
insrc test run --review src/db/search.ts src/db/entities.ts
insrc test run --framework vitest --kind unit src/utils/

insrc test plan <files...>

insrc test plan <files...> [--format <fmt>] [--framework <name>] [--kind <type>] [--cwd <path>]

Generate a test plan without executing. Outputs JSON or markdown.

OptionDescriptionDefault
--formatjson or mdjson
--frameworkOverride frameworkauto-detect
--kindunit or liveunit
